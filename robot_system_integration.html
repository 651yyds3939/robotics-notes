机器人系统集成：数据流与控制流如何闭环

核心定位：单看「感知 / 决策 / 控制 / 执行」任一层都不够——系统集成关心的是谁把什么数据、在什么频率、通过什么中间层交给谁，以及断一环时系统如何降级。

👉 全链路索引：机器人全链路思维导图 · 研发全流程：robot_development_lifecycle.md · 软件管线：robot_software_pipelines.md · 通信机制：ROS 通信原理

一、阅读地图：两层闭环 × 五种集成模式 × 七个经典形态

1.1 两条闭环（所有案例共用）

闭环周期内容
数据闭环传感 → 估计 → 规划/任务 → 指令 → 执行 → 传感世界状态是否被正确感知并执行
控制闭环设定值 → 控制器 → 驱动 → 电机 → 编码器/力反馈每个自由度是否跟得上指令

系统集成 = 在统一时钟与坐标系下，把多条数据闭环嵌套进控制闭环，并处理多速率（千赫兹力矩环 · 数十赫兹策略/视觉环 · 赫兹级任务环）。

1.2 五种集成模式（按「数据怎么流」分类，而非按机型）

模式核心特征典型断点对应章节
Ⅰ 单机单环感知—规划—控制—执行在同一计算单元规划未接控制器、坐标系未对齐§三.1
Ⅱ 双机快慢环慢环任务/感知与快环平衡/力矩分机跨机下发力矩、观测不对齐、断连§三.2
Ⅲ 跨坐标系感知-操作视觉/力觉目标经变换链变成关节指令手眼标定、浮动基座、目标过期§三.3
Ⅳ 多机协同多平台共享环境表征与任务分配地图不一致、路径冲突、调度延迟§三.4
Ⅴ 人在回路主端输入—映射—从端执行—反馈回传时延、尺度映射、安全互锁§三.5

横切模块（不单独构成机型，但每条链路都可能涉及）：多速率 · 断连降级 · 仿真折叠 / Sim2Real

三、五种集成模式（模式详图）

3.1 模式Ⅰ：单机单环

所有计算与通信在同一主机（或实时核）内完成；数据流是一条链，适合先建立闭环直觉。

flowchart TB
    subgraph ONE["同一计算单元"]
        O1["传感驱动"] --> O2["状态估计"]
        O2 --> O3["规划 / 任务"]
        O3 --> O4["运动控制"]
        O4 --> O5["电机驱动"]
        O5 --> O1
    end
环节典型频率数据形态
传感10–200 Hz原始 + 时间戳
估计20–100 Hz位姿、地图、坐标系
规划5–20 Hz路径、速度指令
控制50–500 Hz轮速 / 关节参考
驱动kHz 内环电流 / 力矩

3.2 模式Ⅱ：双机快慢环

慢环（任务、视觉、大模型）与快环（平衡、力矩、安全）分机部署；机间只传低维意图与状态，不传每个力矩周期的高维原始数据。

flowchart TB
    subgraph HOST["慢环主机 · ~1–30 Hz"]
        H1["感知 / 语义 / 任务调度"]
        H1 --> H2["意图包 · 目标位姿 / 路点 / 阶段 / 时效"]
    end

    subgraph LINK["机间链路 · 有线优先"]
        L1["指令下发 + 状态回传 + 心跳"]
    end

    subgraph RT["快环实时机 · ~50 Hz–1 kHz"]
        R1["全身传感融合"]
        R1 --> R2["快环策略 / 平衡"]
        R2 --> R3["全身控制 → 力矩"]
        R3 --> R4["驱动"]
        R4 --> R1
    end

    H2 --> L1 --> R2
    R1 --> L1 --> H1

6.1 多速率分层

层级典型频率运行位置集成要点
任务 / 语义~1 Hz慢环主机只输出目标与阶段
视觉 / 地图~10–30 Hz慢环或同机结果需对齐坐标系与时间戳
策略 / 局部规划~20–50 Hz快环或同机观测维度与训练一致
全身控制~500 Hz–1 kHz快环物理约束与力矩映射
电流 / 力矩内环kHz 级驱动层最高优先级

层间接口：慢→快传递低维、带时效的目标；快→慢回传压缩状态（模式、错误码、关键标量），避免高频原始数据跨机。

6.2 断连与降级

flowchart TB
    Q{"机间或关键链路中断？"}
    Q -->|否| OK["正常：意图下发 + 状态回传"]
    Q -->|是| WD["看门狗超时"]
    WD --> D1["丢弃过期意图"]
    WD --> D2["快环独立保活 · 站立 / 阻尼 / 安全停止"]
    WD --> D3["禁止未确认的新运动"]
    D1 & D2 & D3 --> LOG["本地日志 · 恢复后对齐"]

适用于模式Ⅱ、Ⅳ、Ⅴ及任何跨进程/跨机集成。

6.3 仿真折叠与 Sim2Real

仿真不改变逻辑骨架，只折叠「控制→物理→传感」的实现：

flowchart TB
    subgraph REAL2["实机"]
        RA["传感驱动"] --> RB["估计/规划"]
        RB --> RC["控制"]
        RC --> RD["电机/机械"]
        RD --> RA
    end

    REAL2 ==>|结构等价| SIM2

    subgraph SIM2["仿真 · 结构等价"]
        SA["合成传感"] --> SB["估计/规划 · 同实机"]
        SB --> SC["物理引擎积分"]
        SC --> SA
    end

对齐优先级：规划以上逻辑尽量同构；观测统计、延迟、接触动力学必须单独标定。部署见 边缘模型部署 · RL / Sim2Real。

八、系统集成常见故障模式

现象可能断点涉及模式排查方向
可视化里模型「飞走」坐标系或时间戳Ⅰ–ⅢTF 树、仿真时钟、时钟同步
能规划不能动规划未接控制或未使能Ⅰ启动项、控制器、安全互锁
策略上实机失稳观测与训练不一致Ⅰ、ⅡURDF、关节顺序、输入维度
抓取系统性偏置变换链错误Ⅲ手眼标定、浮动基座补偿
插孔反复顶死力控相位或延迟Ⅲ阻抗参数、控制频率
多车路口死锁调度 / 占用不一致Ⅳ地图版本、优先级策略
主从遥操作抖动映射或时延Ⅴ限速、预测显示、缓冲
上位掉线下位乱动无看门狗Ⅱ、Ⅴ§6.2
编译通过运行缺模块环境未加载通用环境笔记

九、从集成视角读工作空间

按数据流顺序翻工程，而非按目录名：

1. 启动描述 — 哪些进程在跑？仿真还是实机？对应哪种模式？

2. 参数与 remap — 数据通道名、控制器增益是否一致。

3. 机器人模型 — 关节名是否与控制、训练配置一致。

4. 驱动与控制器 — 力矩 / 位置接口是否匹配硬件。

5. 算法源码 — 确认数据流无误后再读实现。

文件类型说明：ROS 工作空间速查。机型实战案例：kuavo-dev-notes 仓库（GitHub 跨库链接）。
