# 🤖 机器人研发全栈路线图与刷题指南

**行业真相：** 机器人的“地基”与“救火队长”。不负责高深算法，但决定了机器人会不会死机、卡顿。不管算法多牛，落地全靠系统工程师。**目前行业需求量极大，且越老越吃香。**

* **硬件交互：** 熟悉基础的软硬件接口，如 CAN 总线、串口通信 (Serial)、UART。
* **性能调优：** 熟练使用 `gdb`, `perf`, `Valgrind` 抓取内存泄漏和 CPU 性能瓶颈。

* **刷题重心：** **C++ 基础与数据结构安全，多线程并发。**

* **战略放弃：** 困难级的动态规划 (DP)、复杂的图论算法。

## 📂 阶段一：C++ 核心基础“精准瘦身”复习

不搞盲目海量死磕，用 2~3 天时间集中攻克行为树项目必须用到的三大 C++ 核心语法。

### 1. 类与多态 (Class & Inheritance)

* **核心概念：** 父类与子类的继承、虚函数（`virtual`）、重写（`override`）、纯虚函数（`= 0`）。
* **为何重要：** 行为树的所有节点（如 Sequence, Action）都是从一个共同的基类 `TreeNode` 派生出来的，必须通过多态机制统一调用 `tick()` 函数。

### 2. 智能指针 (Smart Pointers)

* **核心概念：** `std::shared_ptr`（共享智能指针）与 `std::make_shared`。
* **为何重要：** 行为树在内存中是一个复杂的指针拓扑网络，使用智能指针可以彻底杜绝 Linux 下最致命的内存泄漏与段错误（Segmentation Fault）。

### 3. 动态数组 (标准库容器)

* **核心概念：** `std::vector` 的声明、元素添加（`push_back`）以及基于范围的 for 循环遍历。
* **为何重要：** 行为树中的控制节点（如 Sequence, Selector）需要管理任意多个子节点，必须使用 `std::vector` 进行动态存储。

## 🌲 阶段二：树形思维训练（概念与 LeetCode 刷题）

用 1~2 天时间打通数据结构理论与机器人业务逻辑的桥梁，培养“深度优先遍历”的本能。

### 1. 核心理论对齐

* 明白二叉树的 `left` / `right` 指针在机器人多叉行为树中是如何演变为 `std::vector<std::shared_ptr<TreeNode>> children` 的。
* 理解二叉树的前序遍历、深度优先搜索（DFS）与行为树 Tick 信号向下传递的等价性。

### 2. LeetCode 精选必刷题

* **LeetCode 144. 二叉树的前序遍历 (Easy)：**
* *目的：* 掌握树的递归与遍历机制。前序遍历（根 -> 左 -> 右）正是行为树标准的执行顺序。


* **LeetCode 104. 二叉树的最大深度 (Easy)：**
* *目的：* 建立极其重要的“回溯与状态返回”思维。在行为树里，基层叶子节点将 `SUCCESS` 或 `RUNNING` 状态一层层向上传递，依靠的就是这种底层逻辑。

## 🤖 阶段三：核心实战项目——自研 C++ 轻量级行为树引擎

进入真正的业务开发，用 C++ 重构对 Kuavo 机器人的高层任务调度。

### 1. 最小可行性产品 (MVP) 搭建

* **步骤一：** 在纯 Linux/Ubuntu 环境下，设计 `TreeNode` 虚基类、`SequenceNode` 控制类、以及自定义的 `ActionNode` 动作类。
* **步骤二：** 引入 `NodeStatus` 枚举（`SUCCESS`, `FAILURE`, `RUNNING`），实现核心的 `tick()` 循环流。
* **步骤三：** 实现一个轻量级的 `Blackboard`（黑板），用于上层视觉数据、状态量在不同节点之间的线程安全共享。

### 2. 与 Kuavo 生态链捏合

* 将行为树引擎封装为标准的 ROS2 C++ 节点。
* 行为树中的各个叶子 Action 节点不再写死逻辑，而是通过 C++ 的 ROS2 Client 转换为对 Kuavo 官方底层 SDK 接口或通信话题的调用。

## 🛡️ 阶段四：工业级真机安全部署（“免死金牌”四步法）

针对实验室宝贵的人形机器人资产，严格遵守规范流程，将真机损坏风险控制在安全线内。

### 1. 软件层：构建防御性代码锁

* **限位拦截：** 在 C++ 模块把目标角度发给驱动的前一行，强制加入关节极限角度（`Joint Limit`）检查。
* **突变拦截：** 限制相邻两个控制周期（如 1ms 周期内）的目标量变化率，防止电机发生由于算法突变引起的暴走。
* **姿态断电保护：** 实时读取 IMU 状态，一旦躯干倾斜角度（Pitch/Roll）超过 30 度安全阈值，行为树立马拦截并调用官方紧急断电/去磁函数（E-Stop）。

### 2. 物理与沟通层：逐级实装验证

1. **纯软件仿真：** 先在 Ubuntu 的 Gazebo 环境中运行 C++ 行为树，观察机器人在虚拟世界中是否出现乱动或卡死。
2. **半实物单轴测试：** 让 Kuavo 保持架起状态，只联动测试单颗关节电机，验证 C++ 通信协议、方向和数值无误。
3. **真机“吊挂”首飞：** 将 Kuavo 用承重吊带悬空挂在龙门架上（确保双脚离地 5~10cm）。运行行为树，观察真机四肢联动响应，即使代码死锁或崩溃也不会砸坏硬件。
4. **报备科研损耗：** 部署前主动向导师或负责师兄同步测试计划与安全防御方案，确保流程正规合规。

## 🏎️ 岗位二：运动规划与控制算法工程师 (Planning & Control / PnC)

**行业真相：** 机器人的“老司机”。决定了机器人走得平不平滑、能不能躲开障碍物。这是自动驾驶和工业机器人的绝对核心，**对数学和逻辑思维要求极高**。

### 1. 核心学习路线

* **数学底座：** 线性代数、微积分、**非线性优化**（极其重要，如拉格朗日乘子法、KKT条件）。
* **核心库：** 精通 Eigen 矩阵运算，熟练使用 Ceres、OSQP、Ipopt 等最优化求解器。
* **前端规划 (Routing & Path Planning)：** 基础图搜索算法（A*、Dijkstra、Hybrid A*、RRT*）。
* **轨迹优化 (Trajectory Optimization)：** 贝塞尔曲线、B 样条曲线平滑、Minimum Snap。
* **底层控制 (Control)：** 运动学与动力学建模、PID 控制、LQR、**模型预测控制 (MPC)**。

### 2. LeetCode 靶向刷题

* **刷题重心：** **空间搜索、矩阵运算、逻辑边界处理。**
* **必刷板块：**
* **树与图搜索 (BFS/DFS)**（如：岛屿数量、迷宫寻路）：A* 算法的基础，每天都会用到。
* **优先队列 / 堆**：维护图搜索中的代价列表（Open List）。
* **矩阵与二维数组**（如：螺旋矩阵、旋转图像）：直接对应栅格地图 (Grid Map) 的处理。
* **滑动窗口与双指针**：用于轨迹点的平滑和前瞻距离 (Lookahead) 计算。

## 🧭 岗位三：SLAM 与定位建图算法工程师 (SLAM & Localization)

**行业真相：** 机器人的“制图师”。门槛极高，数学公式最密集的方向。**目前传统视觉/激光 SLAM 赛道极度内卷**，求职要求通常是名校硕士起步+顶会论文。

### 1. 核心学习路线

* **终极数学：** 概率论（贝叶斯估计）、**李群与李代数**（处理三维旋转的必备工具）、多视图几何。
* **核心框架：** 激光 SLAM (LOAM 家族, Cartographer) 或 视觉 SLAM (ORB-SLAM 家族, VINS-Mono)。
* **后端优化：** 图优化理论，熟练使用 g2o、GTSAM、Ceres 构建并求解位姿图 (Pose Graph)。
* **滤波理论：** 扩展卡尔曼滤波 (EKF)、误差状态卡尔曼滤波 (ESKF) 及其在多传感器融合中的应用。

### 2. LeetCode 靶向刷题

* **刷题重心：** **空间几何、图结构、硬核 C++ 底层实现。**
* **必刷板块：**
* **图与拓扑排序**：直接对应后端优化中的图节点关系。
* **矩阵与数学题**：考察三维空间变换的敏锐度。
* **二分查找**：在海量历史位姿中，基于时间戳快速查找与当前帧匹配的数据。

## 🧠 岗位四：强化学习与具身智能工程师 (RL & Embodied AI)

**行业真相：** 机器人的“黑魔法师”。这是目前最火的赛道（如各种人形机器人公司），但在真实世界中存在巨大的 **Sim-to-Real（仿真到现实）鸿沟**。模型在电脑里跑得飞起，落地实机大概率摔跤。

### 1. 核心学习路线

* **语言切换：** 以 **Python** 为绝对主导（训练），但需懂基础 C++（部署）。
* **深度学习底座：** 熟练掌握 PyTorch 或 JAX，Transformer 架构。
* **强化学习算法：** MDP (马尔可夫决策过程)、Q-Learning、DQN，以及目前主流的 Actor-Critic 家族（**PPO**, SAC）。
* **仿真器生态：** 极其熟练地使用物理仿真平台，如 Isaac Gym (NVIDIA), MuJoCo, PyBullet。
* **Sim-to-Real 技术：** 域随机化 (Domain Randomization)、模仿学习 (Imitation Learning)、数据驱动模型预测。

### 2. LeetCode 靶向刷题

* **刷题重心：** 算法思想偏 Pythonic，**数组操作、动态规划与概率论。**
* **必刷板块：**
* **动态规划 (DP)**：强化学习的底层思想就是近似动态规划（如：打家劫舍、最大子序和）。
* **数组与哈希表**（两数之和、多数元素）：在 Python 中快速处理 Tensor 数据的基本功。
* **概率与随机化题目**（如：打乱数组、按权重随机选择）。

## 👁️ 岗位五：感知算法工程师 (Perception & Deep Learning)

**行业真相：** 机器人的“千里眼”。和互联网的 CV 算法岗高度重合。由于大模型和 BEV 技术的普及，纯传统视觉正在被淘汰，**工程部署能力（让庞大模型在破板子上实时跑起来）比写网络结构更重要**。

### 1. 核心学习路线

* **传统视觉与点云：** OpenCV (图像处理基础)、PCL (点云降采样、聚类分割)。
* **核心模型：** 目标检测 (YOLO 系列)、语义分割、3D 目标检测 (PointPillars, VoxelNet)、BEV (鸟瞰图表示)。
* **极速部署 (极其关键)：** 不能只停留在 PyTorch 训练。必须精通 ONNX 导出，使用 **TensorRT / CUDA / OpenVINO** 将模型在边缘计算设备（如 Jetson Orin）上用 C++ 加速推理。

### 2. LeetCode 靶向刷题

* **刷题重心：** **海量数据处理技巧、Python/C++ 双语切换能力。**
* **必刷板块：**
* **滑动窗口与双指针**：处理视频流帧序列的必备技巧。
* **二叉树与并查集**：用于点云聚类（连通分量分析）。
* **基础字符串操作**：处理数据标注文件、解析 JSON/XML。
