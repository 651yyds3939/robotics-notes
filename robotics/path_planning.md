# 路径规划算法 (Path Planning) —— A* / DWA / RRT / PRM

> 👉 **Kuavo 实战**：[3 地图导航](https://github.com/651yyds3939/kuavo-dev-notes/blob/master/kuavo_notes/3.map_navigation.md) · [3.1 官方导航](https://github.com/651yyds3939/kuavo-dev-notes/blob/master/kuavo_notes/3.1official_navigation.md)
>
> **核心定位**：路径规划是机器人从当前位置到目标位置之间**找出一条无碰撞的最优路径**。它是"上层决策"与"底层控制"之间的桥梁——规划器算出"走哪条路"，控制器执行"怎么迈出这一步"。

---

## 第 0 章：路径规划一句话

> **大白话**：全局规划（A*）定路径；局部规划（DWA）实时避障；二者配合完成导航。

---

## 第 1 章：两大分类

| | 全局规划 (Global Planner) | 局部规划 (Local Planner) |
|------|---------------------------|--------------------------|
| 输入 | 静态地图 + 起点 + 终点 | 全局路径 + 实时传感器 + 当前速度 |
| 输出 | 一条从起点到终点的完整路径 | 下一时刻的速度指令 $(v, \omega)$ |
| 频率 | 一次性的，或地图更新时重规划 | 高频（10-50Hz）持续运行 |
| 代表 | A*, RRT, PRM | DWA, TEB, MPC |

## 第 2 章：全局规划算法

### 2.1 A* —— 基于搜索的最优路径

A* 在栅格地图上搜索，用启发式函数引导搜索方向：

$$f(n) = g(n) + h(n)$$

- $g(n)$：从起点走到当前节点的实际代价
- $h(n)$：从当前节点到终点的**估计**代价（启发式，通常用欧几里得/曼哈顿距离）

> **关键**：$h(n) \leq \text{真实代价}$ 保证 A* 一定能找到最优解。$h(n)$ 越接近真实值，搜索越快。

### 2.2 RRT (Rapidly-exploring Random Tree) —— 基于采样的高维规划

A* 在**高维空间**（如机械臂 7 维关节空间）中会因状态爆炸而失效。RRT 用随机采样策略：

```text
1. 在状态空间中随机撒一个点
2. 从现有树中找到离这个点最近的节点
3. 从最近节点向随机点方向扩展一步
4. 扩展出的新点如果没有碰撞，加入树中
5. 重复直到树连接到目标区域
```

> **RRT***：RRT 的优化版，引入"重布线 (rewiring)"步骤，渐进收敛到最优路径。

### 2.3 PRM (Probabilistic Roadmap) —— 先建路网再查询

适合**同一环境多次查询**的场景：离线构建一个概率路网 → 每次查询时在路网上跑 A*。

## 第 3 章：局部规划算法 —— DWA (Dynamic Window Approach)

DWA 是 [ROS Navigation](./ros_logic.md) 中最常用的局部规划器。核心思想：

### 3.1 三步流程

```text
[1] 速度采样：在机器人动力学允许的范围内采样多组 (v, ω) 候选
 约束条件：最大速度/加速度限制 + 安全刹车距离
[2] 轨迹前向模拟：对每组 (v, ω) 前向模拟一小段圆弧轨迹
[3] 代价评分：对每条轨迹打分，选最低代价的指令下发
```

### 3.2 评分函数

$$\text{cost} = \alpha \cdot \text{heading\_cost} + \beta \cdot \text{obstacle\_cost} + \gamma \cdot \text{vel\_cost} + \delta \cdot \text{path\_cost}$$

- **heading_cost**：轨迹末端的朝向与目标方向的夹角（越小越好）
- **obstacle_cost**：轨迹离最近障碍物的距离（越大越好）
- **vel_cost**：速度损失（速度越大越好，鼓励高效）
- **path_cost**：偏离全局路径的距离（越小越好）

### 3.3 DWA 的局限

- 使用圆弧轨迹前向模拟，不适合狭小空间的精细机动
- 评分权重是手工调参，没有理论最优解
- 对人形机器人：DWA 是**差速底盘**的规划器，不直接适用于双足步行

## 第 4 章：人形机器人的路径规划

双足人形的路径规划远比轮式复杂：

| 轮式机器人 | 人形机器人 |
|-----------|-----------|
| 状态空间 3D $(x, y, \theta)$ | 需要同时规划**脚步序列**（每个落脚点的 $(x, y, \theta)$） |
| 规划粒度 = 连续轨迹 | 规划粒度 = 离散的**脚步序列** (footstep planning) |
| 不考虑动力学 | 需要考虑 ZMP / Capture Point 约束保证不摔倒 |

### 常用方案

- **Footstep Planning**：在 2.5D 高程图上搜索落脚点序列，然后用轨迹优化填充分步态
- **[RL-based Locomotion](./RL.md)**：RL 策略直接接收目标速度，把路径规划简化为"告诉 RL 往哪走"——亦与[行走 RL](./RL.md) 正在做的事

## 关键词速查

| 术语 | 解释 |
|------|------|
| **A*** | 启发式搜索，网格地图上的最优路径算法 |
| **RRT** | 随机采样树，解决高维空间路径规划 |
| **PRM** | 概率路网，离线建图+在线查询 |
| **DWA** | 动态窗口法，最常用的局部规划器 |
| **TEB** | 时序弹性带，考虑时间最优的局部规划 |
| **Occupancy Grid** | 占据栅格地图，每个栅格标记为 occupied / free / unknown |
| **Configuration Space** | 构型空间：把所有关节角度当作维度的高维空间 |
