# 机器人系统集成：数据流与控制流如何闭环

> **核心定位**：单看「感知 / 决策 / 控制 / 执行」任一层都不够——系统集成关心的是**谁把什么数据、在什么频率、通过什么接口交给谁**，以及断一环时系统如何降级。
>
> 👉 全链路索引：[机器人全链路思维导图](../robot_system.md) · 算法拓扑：[AI 与机器人学习](./AI_learning_robotics.md) · 工作空间文件：[ROS 文件速查](./doc_concept.md) · 通信机制：[ROS 通信原理](./ros_communication.md)

## 一、集成视角：两层闭环

成熟机器人里同时存在两条闭环：

| 闭环 | 周期 | 内容 |
|------|------|------|
| **数据闭环（Data Loop）** | 传感器 → 估计 → 规划 → 指令 → 执行 → 传感器 | 「世界状态是否被正确感知并执行？」 |
| **控制闭环（Control Loop）** | 设定值 → 控制器 → 驱动 → 电机 → 编码器/力矩反馈 | 「每个关节/轮子是否跟得上指令？」 |

**系统集成** = 把多个子系统的数据闭环嵌套进统一的控制闭环，并处理好**多速率**（1 kHz 力矩 / 50 Hz RL / 30 Hz 视觉 / 1 Hz 大模型）。

## 二、轮式 AGV 示例：四步协同（教学用基准）

以 [ROS 2 框架](./ros_logic.md) 下自主导航轮式机器人为例，完整链路如下。

### Step 1 — 感知上传（Sensing）

| 环节 | 组件 | 输出 |
|------|------|------|
| 本体 | 编码器、IMU | 轮速、角速度 |
| 环境 | 2D LiDAR | `/scan` |
| 驱动 | 串口/USB 节点 | 原始字节 → ROS 消息 |
| 融合前 | HAL / `robot_localization` | `/odom`（可选） |

### Step 2 — 决策规划（Thinking）

| 环节 | 组件 | 输出 |
|------|------|------|
| 定位建图 | [SLAM](./slam.md) + [EKF](./state_estimation.md) | 位姿 + [TF](./tf_tree.md) + `/map` |
| 全局规划 | A* / Navfn | 全局路径 |
| 局部规划 | DWA / TEB | `/cmd_vel`（`geometry_msgs/Twist`） |

👉 详见 [路径规划专题](./path_planning.md)

### Step 3 — 控制解算（Commanding）

| 环节 | 组件 | 输出 |
|------|------|------|
| 通信 | 串口 / CAN | 速度指令到下位机 |
| 运动学 | 差速逆解 | 左右轮目标转速 |
| 闭环 | [PID](./pid_control.md) | 目标电流 / 力矩 |

### Step 4 — 执行做功（Actuation）

| 环节 | 组件 | 输出 |
|------|------|------|
| 驱动 | [FOC](./motor_foc.md) | 三相电流 |
| 机械 | 电机 + 减速器 + 轮子 | 位移 |
| 反馈 | 编码器 | 新一轮 Step 1 |

### 仿真映射

Gazebo 等仿真中 **Step 3–4 被折叠**：`gazebo_ros2_control` 或差速插件直接吃 `/cmd_vel`，物理引擎积分出位姿并**反向生成** `/scan`、`/odom`。理解这层映射是 [Sim2Real](./RL.md) 的基础；部署见 [边缘模型部署](./edge_deployment.md)。

## 三、人形机器人（Kuavo 类）：双机 + 多速率集成

人形系统不是「一条 `/cmd_vel` 走天下」，而是**上位机大脑 + 下位机小脑**分工。

```text
┌─────────────────────────────────────────────────────────────┐
│  上位机（Orin / NUC）  ~ 1–30 Hz 异步                       │
│  视觉 YOLO · VLA/LLM · 导航目标 · 任务状态机               │
│       │ 目标位姿 / 抓取点 / 速度指令 (Topic/Service)          │
└───────┼─────────────────────────────────────────────────────┘
        │ 千兆以太网 · ROS Master 在下位机 · 见 ros_communication
┌───────▼─────────────────────────────────────────────────────┐
│  下位机（胸内 NUC + RT-Preempt）                             │
│  50 Hz  RL 策略 (ONNX) ──► 参考关节 / 落足 / 质心意图         │
│  1 kHz  WBC/MPC ──► 关节力矩 τ ──► EtherCAT ──► 关节 FOC    │
│  100+ Hz 关节 PID / 状态估计 / 触地检测                      │
└─────────────────────────────────────────────────────────────┘
```

| 层级 | 典型频率 | 运行位置 | 集成要点 |
|------|----------|----------|----------|
| 大模型 / VLA 任务 | ~1 Hz | 上位机 | 只输出**意图/目标**，不直接出力矩 |
| 视觉检测 / 深度 | ~30 Hz | 上位机 | 结果经 TF 变换到下位机坐标系 |
| RL 行走 / 舞蹈 | ~50 Hz | 下位机 | [ONNX](./edge_deployment.md) 推理；观测向量与 URDF 顺序必须一致 |
| WBC / MPC | ~500 Hz–1 kHz | 下位机 | 对 RL 输出做物理约束与力矩映射，见 [动力学与控制](./dynamics_control.md) |
| 电流环 FOC | kHz 级 | 关节驱动板 | 见 [FOC 专题](./motor_foc.md) |

**混合级联**：RL 提供「直觉」→ WBC/MPC 做安全兜底 → FOC 执行。任一上层断连，下位机应能**维持站立或安全趴下**（思维导图第七章安全窗口）。

## 四、典型 Topic / 数据流表（概念级）

| 数据 | 方向 | 常见消息 | 消费者 |
|------|------|----------|--------|
| 关节状态 | 驱动 → 全网 | `sensor_msgs/JointState` | 估计、RL、日志 |
| IMU | 下位机 → 估计 | `sensor_msgs/Imu` | EKF、RL obs |
| 图像 / 点云 | 相机 → 上位机 | `sensor_msgs/Image`, `PointCloud2` | 感知、VLA |
| 速度 / 步态指令 | 规划 → 控制 | 自定义或 `Twist` | 底盘 / RL 参考 |
| 关节力矩 / 位置指令 | 控制 → 驱动 | 自定义、`Float64MultiArray` | `ros2_control` |
| TF | 多节点广播 | `tf2` | 视觉抓取、导航、RViz |

话题选型原则见 [ROS 通信原理](./ros_communication.md)（高频用 Topic，配置用 Parameter，一次性查询用 Service）。

## 五、系统集成常见故障模式

| 现象 | 可能断点 | 排查方向 |
|------|----------|----------|
| RViz 里机器人「飞走」 | TF / 时间戳 | [TF 树](./tf_tree.md)、`use_sim_time`、NTP |
| 能规划不能动 | 控制层未收到 `/cmd_vel` 或力矩未使能 | launch 是否启动 controller、`ros2 control list` |
| RL 实机立刻摔 | Sim2Real 观测不对齐 | [URDF](./robot_modeling.md)、关节顺序、`.info` 与 ONNX |
| 编译过、运行找不到包 | 环境未 source | `source install/setup.bash`；见 [环境笔记](./environment.md) |
| 上位机挂了下位机乱动 | 双机集成无看门狗 | 网络断连策略、WBC 独立保活 |

## 六、从集成视角读工作空间

集成调试时建议按数据流翻文件，而不是按目录乱看：

1. **Launch** — 启动了哪些节点？仿真还是实机？  
2. **config YAML** — 话题名 remap、控制器参数。  
3. **URDF/xacro** — 关节名是否与控制器、RL 训练一致。  
4. **驱动 / `ros2_control` 配置** — 力矩/位置接口是否匹配硬件。  
5. **源码** — 最后才读算法细节。

各文件类型说明见 [ROS / Ubuntu 工作空间文件速查](./doc_concept.md)；Kuavo 真机联调案例见 [kuavo-dev-notes 仓库](https://github.com/651yyds3939/kuavo-dev-notes)（GitHub 跨库链接）。

## 七、与思维导图九层的关系

| 思维导图层级 | 在集成中的角色 |
|--------------|----------------|
| 感知层 | 数据闭环输入 |
| 决策层 | 任务与路径；不直接碰电机 |
| 控制层 | 多速率控制核心；WBC/RL/PID |
| 执行层 | FOC + 电机 |
| 通信层 | Topic 与双机网络；断连策略 |
| 安全层 | 急停、状态切换窗口、限位 |
| 工程化 | Docker、Git、仿真与 Sim2Real 流水线 |

把本篇当作「**竖切一条数据流**」；思维导图是「**横切全栈地图**」。两者对照阅读效果最好。
