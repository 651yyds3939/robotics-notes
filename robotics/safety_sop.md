# 人形机器人真机安全 SOP (Safety Standard Operating Procedure)

> **核心定位**：人形机器人（29-DOF 双足）真机调试具有极高的物理风险——关节爆冲、摔机、电池甩落均可能导致数月的返厂维修甚至人身伤害。本章沉淀了在 Kuavo 4 Pro 上数百次真机点火后总结的安全铁律，不是书本理论。
>
> 👉 实战：[RL 行走 Sim2Real 安全部署](https://github.com/651yyds3939/kuavo-dev-notes/blob/master/kuavo_notes/15.4RL_lab_sim_to_real.md) · [舞蹈真机安全红线](https://github.com/651yyds3939/kuavo-dev-notes/blob/master/kuavo_notes/23.1.RL_dance_overview.md)

## 第 0 章：安全哲学

> 人形机器人是一个非线性、欠驱动、不稳定的倒立摆。一旦算法下发了超越支撑域的指令，重力会在极短时间内将其砸向地面。**最高优先级永远是：龙门架绝对不解绳、动作维持极低动态、现场时刻具备物理急停能力。**

## 第 1 章：物理安全防线

### 1.1 龙门架与安全绳

| 步骤 | 操作 | 原理 |
|------|------|------|
| 降落 | 摇动绞盘让机器人双脚平整踩地 | 触发底部触地开关（20N 阈值） |
| 微松 | 贴地后**最多再放 2-3cm** 绳子 | 钢丝绳微弯但不松垮，触发触地判定 + 保留极限防坠 |
| **红线** | 绝对不允许完全解绳 | 机器人自由站立时摔倒无人能救 |

### 1.2 急停链路

```
48V 动力电 ──→ 急停开关 ──→ 电机驱动器
低压逻辑电 ──→ (不经过急停) ──→ 主板/NUC/Orin/网络
```

> **关键设计**：急停仅切断高压动力电，低压逻辑控制电**不受影响**——拍下急停后仍能记录故障日志、维持网络通信。

### 1.3 人员禁区

| 区域 | 风险 | 规则 |
|------|------|------|
| 机器人正前方 | Pitch 俯仰方向劈叉前摔 | **严禁站人** |
| 机器人正后方 | Pitch 仰倒后摔 | **严禁站人** |
| 侧后方 45° | 相对安全 | 操作员站位，双手轻扶肩膀校准 IMU |

### 1.4 双人操作制度

| 角色 | 职责 |
|------|------|
| **主操作员** | 扶正躯干、操作遥控器、执行起立/RL 切换 |
| **安全员** | 手悬急停按钮、盯紧关节异常啸叫、0.1s 内切断 |

## 第 2 章：起立与接管 SOP（黄金 3 秒）

### 2.1 遥控器初始状态

- **E 档**：最左（安全锁）
- **F 档**：最右（待机）

### 2.2 点火时序

```text
T-0.0s:  安全员手悬急停，主操作员扶正躯干（校准 IMU 零点）
T+0.5s:  主操作员拨 F 档至左 → 经典 WBC 低刚度起立（无爆冲，棉柔站起）
T+3.0s:  确认站立平稳、松开双手（断开外部干扰）
         按下 B 键 → RL 神经网络接管
```

### 2.3 遥控器按键速查

| 按键 | 功能 |
|------|------|
| **D** | STANCE（站立锁死） |
| **B** | 切入 RL 控制（ONNX 推理接管） |
| **E 档** | 急停断电（最左 = 安全锁） |
| **F 档** | 起立/站立切换（左=起立，右=待机） |

## 第 3 章：分级安全响应

### 3.1 软件保护层

| 保护 | 机制 |
|------|------|
| **关节软限位** | 动态限制最大角度/速度/力矩 |
| **碰撞检测** | 关节力矩超过阈值 → 自动降刚度（变软） |
| **看门狗** | 控制节点死机 → 自动锁死所有关节 |
| **异常状态机** | 故障分级：Warning → Limp → Emergency Stop |

### 3.2 硬件保护层

| 保护 | 机制 |
|------|------|
| **电磁抱闸** | 断电自动锁死关节，防止断电瞬间摔落 |
| **机械限位挡块** | 关节物理限位，软件限位失效的最后防线 |
| **快熔断器** | 短路/过流瞬间切断，保护电调和电池 |

## 第 4 章：Sim2Real 特有的安全风险

### 4.1 起立阶段的阶跃冲击

经典 WBC 起立和 RL 接管之间存在**控制信号切换**。如果 RL 策略的第一帧输出和 WBC 最后一帧力矩差异过大（阶跃），关节会瞬间爆冲。

> **对策**：WBC 起立用**低刚度**，RL 切入后用**渐进加载**（力矩从 0 线性过渡到 RL 输出，而非瞬间跳变）。

### 4.2 观测空间对齐错误

ONNX 模型的输入维度必须和训练时逐位对齐。如果 C++ 侧 `humanoidController.cpp` 给错了一维观测，RL 策略收到的是垃圾向量，输出可能是随机暴力动作。

> **对策**：ONNX 导出后先在 MuJoCo Sim2Sim 中验证（同输入 → 同输出），再上真机。

### 4.3 触地判定失效

如果安全绳绷太紧，机器人被吊着，脚底触地开关无法触发。RL 策略以为自己在飞，输出在空中乱踩的动作。

## 关键词速查

| 术语 | 解释 |
|------|------|
| **龙门架** | 铝型材框架 + 电动绞盘 + 安全绳，真机调试必备 |
| **急停 (E-Stop)** | 物理断开 48V 动力电，保留低压逻辑电 |
| **E/F 档** | H12 遥控器拨杆，E=安全锁，F=起立/站立切换 |
| **黄金 3 秒** | WBC 起立 → RL 接管的安全切换窗口 |
| **降刚度** | 关节遇异常阻力自动变软，防止硬件损坏 |
