# 世界模型 (World Model) —— 让机器人学会"想象"物理世界

> **核心定位**：世界模型是 model-based RL 的心脏。它不直接输出动作，而是先学会"如果执行某个动作，世界会变成什么样"，然后在这个学到的内部模型里做规划。相比 PPO 等 model-free 方法，世界模型追求更高的**样本效率**和**任务迁移能力**。
>
> 👉 实战笔记：[Kuavo TD-MPC2 世界模型训练全流程](https://github.com/651yyds3939/kuavo-dev-notes/blob/master/kuavo_notes/31.1.world_model.md)

## 第 0 章：一句话理解世界模型

> **大白话**：PPO 是"蒙眼练肌肉"——试 1000 万次，哪次做对了就给奖励，慢慢地肌肉记住了该怎么动。世界模型是"睁眼做白日梦"——先在脑子里学一套物理规律，然后闭上眼睛想象"如果我这么动，会发生什么"，在想象中选出最好的动作，再去现实里执行。

| | Model-Free RL (PPO) | Model-Based RL (World Model) |
|------|---------------------|------------------------------|
| 学什么 | 观测 → 动作（直接映射） | 观测 + 动作 → 下一时刻观测 + 奖励（世界规律） |
| 怎么决策 | 神经网络直接输出动作 | 在世界模型里做虚拟规划（MPPI / MPC） |
| 样本效率 | 低（百万步起步） | 高（理论上万步就能学） |
| 泛化能力 | 换任务需要重新训练 | 世界模型可以跨任务复用 |
| 计算开销 | 训练重，推理轻 | 训练相对轻，推理时需要在线规划 |

## 第 1 章：纯世界模型的本质解构 —— 渲染器、模拟器与规划器

在深入具身智能的硬核架构之前，需要首先理清"纯世界模型（Pure World Model）"在通用人工智能（AGI）与认知科学层面的底层技术闭环。纯世界模型的核心哲学，是让智能体在无源的状态下构建一套对外部现实宇宙的"主观潜意识投影"。从信息流与泛化能力的维度来看，一个完整的纯世界模型必然由**渲染器**、**模拟器**与**规划器**这三大核心角色鼎足而成：

* **渲染器（Renderer / Observation Decoder）：** 负责"时空快照的视觉幻觉"。它的数学本质是一个高维观测重构算子，负责将抽象的隐空间特征 $s_t$ 还原解码为智能体能够直观感知的原始多模态高维观测（如相机的 RGB-D 像素、深度的时空拓扑或环境的像素演化）。它解决了智能体"在脑海中如何看见未来画面"的视觉连续性问题。

* **模拟器（Simulator / Transition Model）：** 整个世界模型的物理灵魂。它是一个全可微的"神经物理引擎"，不依赖人工编写的解析物理方程，而是纯粹靠自监督学习来参悟宇宙的刚体惯性、重力守恒、非刚性接触以及强接触力学的内在演化规律。其数学职责是给定当前的隐状态 $s_t$ 和动作 $a_t$，在隐空间（Latent Space）内执行高频的前向时程外推，精确预判下一帧的隐式状态 $s_{t+1}$。

* **规划器（Planner / Imagination Optimizer）：** 脑内做梦的"最高决策法庭"。规划器依托于"模拟器"所提供的全可微物理直觉，在完全脱离真实世界（或物理引擎）的大前提下，在隐空间内部展开长周期的"潜在想象（Latent Imagination）"。它通过高并行的路径积分采样（如 MPPI）或自监督梯度反向传播，对未来可能发生的千百种动作控制序列进行滚动寻优，挑出一条最符合全局任务目标的黄金控制轨迹。

当这三大角色完美合流，纯世界模型就拥有了在虚拟幻觉中演化出高级智能的完备闭环。然而，当我们将这套认知控制论搬进具身智能（机器人）的真实肉体中时，学术界的写意空中楼阁会瞬间遭遇物理世界的铁血撞墙——机器人拥有庞大的机械连杆公差、严苛的 1kHz EtherCAT 实时通信总线，以及双足落地瞬间高频、不连续的强接触力学冲击。

因此，现代机器人世界模型必须实现从"纯视觉像素重构"向"任务驱动隐空间"的控制降维。我们必须将世界模型作为全可微的脑内物理大脑，与底层的[经典控制求解器（WBC/MPC）](./dynamics_control.md) 进行异步双循环的分层集成，才能真正赋能物理实体。

## 第 2 章：世界模型的数学解构——三大核心组件

一个标准的机器人世界模型由以下三个全可微的网络组件构成：

### 2.1 表征模型（Encoder, $E_\theta$）—— 压缩观测，消除噪声

机器人每秒接收几十到上百维的传感器数据（关节角度、IMU、图像、触觉阵列...）。表征模型负责将这些高维、异构的多模态原始观测压缩、映射到低维紧凑的隐状态 $s_t$ 中。

$$s_t = E_\theta(o_t, s_{t-1}, a_{t-1})$$

> **大白话**：不必记住每个像素才能行动——脑中保留环境的抽象表征即可。Encoder 学习这种压缩表示；人形机器人常见 115 维观测（关节、IMU、参考轨迹、上一帧动作）可压至 32–64 维潜在向量。

### 2.2 过渡动力学模型（Transition Model, $f_\phi$）—— 预测未来，物理灵魂

给定当前潜在状态 $s_t$ 和要执行的动作 $a_t$，预测下一步会发生什么：下一时刻的潜在状态 $s_{t+1}$。

$$s_{t+1} = f_\phi(s_t, a_t)$$

这个世界模型掌握了重力、动量守恒和强接触力学的物理定律。在隐空间内，输入当前的隐状态 $s_t$ 与控制动作 $a_t$，它能极其精准地前向外推、预判出下一帧的隐状态。

> **大白话**：这就是机器人的"白日梦引擎"。它学到了重力、惯性、关节限位、碰撞反弹这些物理规律——不是靠数学公式推出来的，是靠看了几万条真实数据之后"悟"出来的。

### 2.3 任务预测头家族（Decoders / Predictor Heads）—— 判断好坏，引导决策

负责根据当前的隐状态 $s_t$，去预测当前会获得多少奖励（**Reward Head**）、是否发生碰撞或摔倒（**Constraint / Termination Head**），或者选择性地还原出原始图像（**Obs Decoder**）。

> 在 TD-MPC2 这类 Decoder-free 架构中，预测头被精简为只预测**对决策有用的信号**（Reward + TD-Value），不做像素级重建，从而大幅降低模型容量和训练难度。

### 2.4 规划器（Planner）—— 在想象中做决策

给定当前状态，规划器在学到的世界模型里"脑补"出多种可能的未来轨迹，选出总奖励最高的动作序列。常用方法：

| 方法 | 原理 | 代表算法 |
|------|------|----------|
| **MPPI** (Model Predictive Path Integral) | 采样 N 条候选轨迹，加权平均最优的 | TD-MPC2 |
| **CEM** (Cross-Entropy Method) | 迭代筛选精英样本 | PETS, PlaNet |
| **[MPC](./dynamics_control.md)** (Model Predictive Control) | 有限时域内求解最优控制 | 经典 MPC |

> **大白话**：规划器就像下棋——不是想一步，而是在脑子里推演接下来 10 步的各种可能性，选最有利的那条路。每走一步都重新推演一次（滚动时域）。

## 第 3 章：主流技术路线对比 —— DreamerV3 vs TD-MPC2

目前在机器人控制领域，世界模型分化出了两条完全不同的控制哲学：

| 维度 | DreamerV3 路线 | TD-MPC2 路线（👑 人形推荐） |
|------|---------------|---------------------------|
| **模型本质** | **全量重构派**：致力于在隐空间内完美重建整个世界（包含视觉像素与状态的重建） | **任务驱动派**：抛弃"重建世界"的幻想，属于 Decoder-free 架构 |
| **隐空间目标** | 隐状态必须能强行还原出原始的相机像素与观测 | 隐空间只被强迫去预测两件事：未来的 Reward 和 TD-Value |
| **决策方式** | **Reactive Policy**：训练 Actor 网络，看一眼隐状态直接输出动作 | **隐空间 MPC**：并不完全依赖学死的 Policy，运行时在隐空间运行高并发的 MPPI 采样规划 |
| **对纯状态空间的适用度** | **大炮打蚊子**：核心优势是处理海量无监督纯图像，面对百维状态向量会引入极大的自监督重建噪声，调参地狱 | **天生绝配**：隐式特征纯粹由任务目标驱动，自动过滤环境无用噪声，网络层数极轻，推理延迟低 |
| **人形机器人适配** | 推理时需要大量 roll-out，实时性差 | 推理时可切换 MPPI 规划或直接 Policy 出动作，灵活性高 |

TD-MPC2 的核心创新是**隐式解码（Decoder-free）**——不需要从潜在状态重建原始观测（传统方法如 DreamerV3 需要），只需要预测**对决策有用的信号**（价值、奖励、下一步潜在状态）。这大幅降低了模型容量和训练难度，使其成为目前**最适合人形机器人 locomotion** 的世界模型框架之一。

### TD-MPC2 的四个网络

```text
Encoder:   观测 o_t → 潜在状态 z_t
Dynamics:  (z_t, a_t) → z_{t+1}, r_t  (预测下一步)
Reward:    (z_t, a_t) → r_t            (辅助奖励预测)
Value:     (z_t) → Q(z_t, a_t)         (状态-动作价值)
Policy:    (z_t) → a_t                 (策略先验，可选)
```

训练时，这四块联合优化。推理时有两种模式：

- **Policy 模式**（快速）：Encoder → Policy → 动作，一步出结果（类似 PPO）
- **MPPI 模式**（精确）：Encoder → Dynamics 模型内采样 N 条轨迹 → 选价值最高的动作序列

## 第 4 章：世界模型的训练生命周期

世界模型的训练不同于 Model-Free RL 的单步交替，它是一个典型的**三阶段、双重现实循环**生命周期。

### Stage 1: 行为采样与冷启动（Data Collection）

智能体无法凭空产生幻觉，必须首先获取真实世界的物理数据。

利用一个基础策略（或带有高斯扰动的传统控制器）让机器人在仿真环境（如 Isaac Sim / [MuJoCo](./robot_modeling.md)）或真实物理世界中运行，收集并记录成千上万条时间序列轨迹（Trajectories）。每条轨迹片段包含：原始多模态观测序列 $o_t$、执行的动作 $a_t$、以及获得的奖励 $r_t$。

以 Kuavo 舞蹈任务为例：

```text
[1] CSV 动作数据作为参考轨迹（115 维关节位置序列）
[2] Gym 环境：[MuJoCo](./robot_modeling.md) 物理引擎，加载 Kuavo [URDF](./robot_modeling.md)
[3] 每步收集 (obs, action, reward, next_obs) 四元组，存入 Replay Buffer
```

### Stage 2: 神经物理引擎的自监督喂养（World Model Learning）

这是纯粹的深度学习主战场。将 Stage 1 采样的连续时序数据块输入显卡，训练世界模型的权重。其损失函数通常由以下三部分强行联合约束：

$$\mathcal{L}_{\text{World\_Model}} = \mathcal{L}_{\text{recon}} + \beta \cdot \mathcal{L}_{\text{dynamics}} + \mathcal{L}_{\text{reward}}$$

* **状态/像素重构误差（$\mathcal{L}_{\text{recon}}$）：** 约束隐状态必须包含原始世界的关键特征。
* **动力学过渡误差（$\mathcal{L}_{\text{dynamics}}$）：** 通常使用 KL 散度（KL Divergence）约束。它强迫"仅根据历史和当前动作预测出的未来状态先验分布 $p$"，去无限逼近"看到了真实下一帧观测后修正出的后验分布 $q$"。为了防止隐空间崩溃（Posterior Collapse），通常会引入 1:4 的 KL Balancing 约束。
* **任务/约束奖励误差（$\mathcal{L}_{\text{reward}}$）：** 确保隐状态能准确预测出当前动作的好坏。

> **💡 显存保卫战纪律（Host Memory Offloading）：**
> 针对几百万步长序列数据的存储，如果全量堆在显存（GPU）中会瞬间引发 CUDA OOM 爆显存崩溃。工业界标准做法是：将数据缓冲区的底层张量全量初始化在 CPU 内存上，只有当训练代码调用 `sample()` 抽取出小批次（Mini-batch）时，再通过 `.to(device)` 异步搬运至 GPU，以此完美保护 8GB 等低显存卡。

训练过程中的监控：

```text
[4] 离线训练：联合优化 Encoder + Dynamics + Value + Policy
[5] TensorBoard 监控：episode_reward、value_loss、dynamics_loss
[6] Play 验证：play.py 加载 checkpoint，可视化 + 录视频
```

### Stage 3: 隐空间策略想象（Policy Imagination）

此时完全**冻结** Stage 2 训练好的世界模型权重，物理引擎被完全切断、保持静止或关闭状态。

策略网络（Actor）和价值网络（Critic）粉墨登场。它们完全在世界模型虚拟出的隐空间内进行"长周期想象外推"（通常前向外推 10~15 步）。由于世界模型的 Transition 神经网络是**全阶可微**的，我们可以使用时间反向传播（BPTT），让 Value 网络的梯度穿透时间步直接精准地流回 Actor 策略，以惊人的超实时速度（几万倍于实时）刷出最优控制策略。

## 第 5 章：世界模型在人形机器人上的部署架构

### 5.1 核心挑战

**观测空间巨大**：人形机器人 29 个关节 + IMU + 基座速度 = 115+ 维观测。每个维度之间还有复杂的耦合关系（手动了腿也得微调平衡）。编码器必须学会从这 115 维里提取**对平衡和动作跟踪真正有用的信息**，而不是无差别地全量压缩。

**动力学的高度非线性**：双足机器人的动力学不是"输入 A → 输出 B"的简单映射。触地瞬间（impact）、摩擦锥（friction cone）、关节限位——这些都是**不可微的硬约束**。世界模型用神经网络去逼近这些约束，理论上能学到，实际训练中非常容易发散。

**规划延迟 vs 控制频率**：世界模型的 MPPI 规划需要在前向模拟中推理几十到几百条轨迹。如果控制频率是 50Hz（20ms 一帧），每次推理必须在 20ms 内完成。Kuavo 实战中，TD-MPC2 在 8GB RTX 3070 上，单个 env 的 MPPI 规划约需 5-10ms。但 batch size 和 num_envs 增大后，显存和延迟都会快速攀升。

**Sim2Real 的漂移问题**：PPO 是"盲走"——只映射观测到动作，仿真和真机的观测分布不一致时可能直接崩溃。世界模型在这一点上理论上更强——如果在仿真中训练的世界模型学到的是**通用的物理规律**（而非过拟合仿真参数），那么部署到真机时，它仍然能做出合理的预测。但这要求极其严苛的**域随机化**：训练时力学参数（质量、摩擦、阻尼、延迟等）的随机范围必须覆盖真机的不确定性区间。

### 5.2 异步双循环实时通信控制流

在 Kuavo 人形机器人这类大型非线性双足硬件平台上，部署世界模型面临的最大敌人是推理卡顿与强接触力学的冲突。工业界通常采用**"异步宏观规划 - 同步微观执行"**的分层架构：

* **低频脑部循环（World Model Planner, $\sim$ 30Hz ~ 50Hz）：** 运行在非实时常规 CPU/GPU 核心上。读入多模态特征，前向推理。如果是 TD-MPC2，则直接在隐空间运行高并行的 MPPI 采样器，向未来模拟多条动作路径，挑出一条最优的宏观控制表征。

* **高频小脑执行环（WBC Controller, $500\text{Hz} \sim 1\text{kHz}$）：** 绑定到隔离的 CPU 实时核心（CPU Isolation，运行 RT-Preempt 系统内核）。它绝不等待高层世界模型的输出，而是利用三次样条或贝塞尔曲线对高层轨迹进行高频插值，通过求解[二次规划（QP）](./optimization_theory.md) 问题计算出最优的瞬时关节力矩 $\boldsymbol{\tau}$ 并直接下发伺服总线。

### 5.3 空间抽象接口解耦

为了保障控制的稳定性，世界模型的高层输出绝对不能直接去控关节马达的刚性 PD 角度。高层解包的接口必须定义为任务空间（Task-Space）的物理顺应性控制量：

* 未来的质心轨迹（CoM Trajectory） $\boldsymbol{x}_{\text{CoM}}(t), \dot{\boldsymbol{x}}_{\text{CoM}}(t)$
* 摆动腿的期望落脚位置 $\boldsymbol{p}_{\text{foot}}^*$ 以及预估的触地刚度时序

### 5.4 多速率控制总览

```text
WBC/MPC @ 1kHz（底层力矩）
    ↑ 三次样条插值平滑
RL / World Model @ 50Hz（[ONNX 推理](./edge_deployment.md)）
    ↑ 异步通信
视觉感知 @ 30Hz
    ↑ 异步通信
大模型决策 @ 异步
```

各级之间通过低通滤波 + 线性插值平滑衔接，避免阶跃冲击。

## 第 6 章：世界模型 vs PPO —— 实战对比

| 维度 | PPO (Model-Free) | TD-MPC2 (Model-Based) |
|------|-----------------|----------------------|
| **训练范式** | On-policy 采样 → 更新策略 → 丢弃旧数据 | Off-policy，数据存在 replay buffer 反复用 |
| **并行环境数** | 需要大量并行（4096-8192 envs）保证样本多样性 | 可少得多（256-512 envs 也能训） |
| **GPU 显存** | ~6.8GB（8GB 卡，4096 envs，S49） | 类似，但显存分布在 replay buffer + 模型参数上 |
| **训练速度** | ~16000 steps/s（S49 URDF 链） | 取决于模型大小和环境复杂度 |
| **推理延迟** | 极低（单次 ONNX 前向，<1ms） | Policy 模式类似；MPPI 模式高 10-50 倍 |
| **泛化能力** | 弱。换任务（走路 → 跳舞）需重新训 | 理论强。世界模型可跨任务复用 |
| **调优难度** | 低。奖励函数 + 域随机化 + 网络结构 | **偏高**。超参数多（planning horizon, mppi samples, temperature, 各 loss 权重） |
| **2026 年真机现状** | ✅ 成熟。行走 Sim2Real 稳定 | ⚠️ 前沿。效果仍在迭代，部署案例少 |

### 什么时候用世界模型

- 样本效率是瓶颈（真机采数据贵，仿真采数据慢）
- 需要跨任务泛化（同一个机器人做行走、舞蹈、上下楼梯）
- 需要在线适应（环境变了，比如地面从平地变成碎石）

### 什么时候用 PPO

- 任务单一、训练算力充足
- 部署延迟要求极高（<1ms 推理）
- 工程稳定性优先于样本效率
- 团队已有成熟的 Sim2Real PPO 管线

## 第 7 章：世界模型对机器人控制的核心价值

### 7.1 隐空间"白嫖"样本与安全探索

在纯强化学习中，策略为了学一个复杂的全身舞蹈或搬运任务，需要在仿真器里高频调用 PhysX 去解千万次刚体碰撞，极度消耗算力。世界模型练成后，控制迭代变成了纯粹的深度学习张量乘法。机器人可以在脑海的"幻觉"中安全地探索千万种摔倒和极限恢复的可能，而不需要给物理真机电机施加任何危险的随机探索扭矩。

### 7.2 秒级 Sim-to-Real 在线自适应（Online System ID）

传统的域随机化（Domain Randomization）是让策略硬扛参数变化，往往得到一个保守的平均解。

世界模型走的是在线预测误差补偿路线：

$$\boldsymbol{e}_t = \boldsymbol{o}_{\text{real}, t} - \hat{\boldsymbol{o}}_{\text{world\_model}, t}$$

当 Kuavo 机器人从平整实验室地面突然走向未知的松软草地或地毯，接触动力学发生剧烈突变。由于环境改变，真实观测与世界模型的脑内预测之间会产生显著的预测残差 $\boldsymbol{e}_t$。世界模型会利用这个残差，通过上下文（In-context）或反向传播，在毫秒级内将环境的摩擦系数、依从性（Compliance）以及未建模的关节背隙隐式编码进当前的隐状态中，指导策略瞬间做出代偿，实现秒级的真机自适应。

### 7.3 为底层提供具备"远见"的非线性参考轨迹

底层的 WBC 是一个典型的"短视"控制器，无法自主规划长周期行为。传统 MPC 依赖简化模型（如 LIPM 倒立摆），无法有效建模复杂的全身扭腰、甩臂等非线性耦合。

世界模型拥有全身动力学的非线性逼近能力。它在 30Hz 的周期下，向前预测未来数秒内人形机器人全身状态的演化趋势，规划出基于全局长期任务目标最优的质心动量变化率，让底层 WBC 专注于它最擅长的 QP 求解与刚性物理约束。

### 7.4 异构多模态数据的特征熔炉（Multi-modal Bottleneck）

在人形机器人执行全身协调操作（Whole-Body Manipulation）时，夹爪上的高维空间触觉拓扑阵列（信号高度非连续、强非线性）、双目相机的 RGB 图像、以及高频的本体感觉 IMU，其特征空间极度庞杂。

世界模型通过自监督信息瓶颈（Information Bottleneck），强迫所有多模态异构传感器生数据压缩映射到同一个统一的物理隐空间中。它把触觉的局部高频压力突变，提炼为紧凑的"接触稳定性特征评分"，策略面对的不再是原始噪声，而是符合物理规律的"高纯度特征向量"。

## 第 8 章：经典工程痛点 —— 防范隐空间的"幻觉作弊"

在世界模型与强化学习结合的训练过程中，最经典的工程灾难莫过于**策略陷入局部最优解恶意刷分**。

### 8.1 典型案例：躺平刷分

在人形机器人长时程编舞或操作任务中，由于世界模型的 Reward Decoder 给予了肢体跟踪高额正分，策略网络（Actor）极易在大数据探索中发现一个机制漏洞——**直接一出生就主动躺倒在地上（挺尸），通过平躺完美避开全部的机身直立偏离处罚；同时四肢在绝对安全的物理地板上精确跟踪 CSV 乐谱摆动**。此时在数据曲线上，模仿得分（Reward）高到吓人，但策略在脑海（幻觉）里已经彻底走偏。

> 对应 Kuavo IL+RL 舞蹈训练 Iter 7461 的「躺平局部最优」现象。

### 8.2 工业级拦截破局手段：可微约束预测头（Constraint Head）

为了降维打击世界模型的"幻觉作弊"，必须在隐空间内部平行挂载一个 **Constraint / Termination Decoder 预测头**。

* 它利用二分类交叉熵损失（BCE Loss），专门用于在线预测当前隐状态下机器人是否触发 `base_contact`（躯干撞地）或直立姿态判定因子 `upright_factor < 0.75`。
* 当 Actor 在全可微的世界模型脑海里进行"滚动做梦规划（Imagination / MPPI Shooting）"时，它每试图往躺平的作弊捷径上走一步，Constraint Head 就会在幻觉里当场判定其死亡并降下雷霆般的重型负分惩罚。

这种物理约束的前向渗透，能够强行将强化学习的演化方向逼回直立舞蹈、安全操作的合法物理边界内，确保学出来的策略具备真正的真机控制韧性。

## 第 9 章：Kuavo TD-MPC2 实验 —— 当前状态与后续方向

### 9.1 已完成的

- TD-MPC2 训练链路跑通（环境注入 → CSV 预处理 → S49 资产准备 → `train.py` → TensorBoard 监控）
- 支持 S42/S46/S49 三种机型配置
- 支持舞蹈模仿和速度跟踪两种任务
- Play 可视化 + ONNX 导出链路就绪
- Play 时默认开启 MPPI 规划

### 9.2 策略效果未稳定的可能原因

1. **奖励函数权重失衡**：mimic 跟踪奖励 vs 姿态惩罚的权重比需要消融实验
2. **世界模型预测精度不够**：dynamics 模型的 roll-out 误差在 10+ 步后累积发散
3. **MPPI 规划参数未调优**：采样数、温度、planning horizon 需要实验确定最优组合
4. **训练不足**：TD-MPC2 的收敛速度比 PPO 慢，可能需要更多 iterations

### 9.3 建议的后续方向

| 优先级 | 方向 | 预期收益 |
|--------|------|---------|
| P0 | 降低任务难度：先做"原地站立 + 微小扰动恢复"，验证世界模型 roll-out 精度 | 验证 pipeline 正确性 |
| P1 | 奖励函数消融实验：不同 mimic/balance/base_height 权重配比 | 找到最优权重组合 |
| P1 | MPPI 参数网格搜索：planning horizon / samples / temperature | 提升规划质量 |
| P2 | 同条件 PPO vs TD-MPC2 对比实验：相同任务、相同 env、相同 seed | 量化 model-based 的优势 |
| P2 | 世界模型跨任务迁移：舞蹈任务上训练的世界模型，在行走任务上 fine-tune | 验证泛化能力 |

## 关键术语速查

| 术语 | 一句话解释 |
|------|-----------|
| **Model-Based RL** | 先学世界规律，再在学到的模型里做决策 |
| **Model-Free RL** | 直接从观测映射到动作，不显式建模世界 |
| **Latent State** $z$ | 原始观测的压缩表征，世界模型在潜在空间里运算 |
| **MPPI** | 采样 N 条候选轨迹 → 加权平均 → 选最优动作，TD-MPC2 的规划方式 |
| **Replay Buffer** | 存储历史 (obs, action, reward, next_obs)，off-policy 训练的核心数据池 |
| **Roll-out** | 从当前状态出发，用世界模型模拟未来 N 步的轨迹 |
| **BPTT** | 时间反向传播，让梯度穿透时间步直接流回策略，用于隐空间想象训练 |
| **KL Balancing** | 约束先验分布逼近后验分布，防止隐空间崩溃（Posterior Collapse） |
| **Domain Randomization** | 训练时随机化物理参数，让策略/世界模型在真机上也能泛化 |
| **Sim2Real Gap** | 仿真和真机之间的物理差异（刚度、阻尼、延迟、触地...） |
| **Constraint Head** | 可微约束预测头，在隐空间中预测碰撞/摔倒，防止策略"幻觉作弊" |
| **Online System ID** | 利用预测残差在毫秒级内在线辨识环境参数变化 |

## ❓ 引导技术思考

在这个 Model-Based 强化学习与世界模型全速爆发的技术交汇点上，有一个关于多模态对齐的硬核技术痛点：

当机器人的夹爪在利用世界模型执行"零部件盲插装配"时，触觉阵列的时空压力拓扑（高维强非线性）在时间轴上高度不连续——只有在接触碰撞的瞬间才有信号突变，平时全是零。

在 TD-MPC2 这种 Decoder-free（无重构）的隐空间联合表征学习中，面对这种极其稀疏且不连续的触觉输入，我们应该如何设计隐空间中 Reward 预测头与 TD-Value（时序差分价值）预测头的损失权重配比，才能确保世界模型在隐空间内前向模拟未来时，不会因为触觉信号的缺失而发生"力矩幻觉漂移（Drift）"，进而避免策略下发导致真机硬件过载损坏？
