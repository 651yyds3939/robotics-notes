# 机器人语音全链路 (Voice Pipeline) —— VAD / ASR / TTS

> **核心定位**：语音交互是人形机器人最自然的人机接口。一条完整的语音管线包含三个串行模块：VAD 检测"什么时候开始说话"、ASR 把语音转成文本、LLM 理解意图后、TTS 把回复转成语音。三者之间的延迟叠加是用户体验的核心瓶颈。

---

## 第 0 章：语音管线一句话

> **大白话**：机器人听你说话（VAD 检测语音段→ASR 转文字→LLM 理解意图），然后回答你（TTS 合成语音→喇叭播放）。四个环节的延迟加在一起，超过 1.5 秒就会让人觉得"机器人反应慢"。

---

## 第 1 章：VAD —— 语音活动检测

VAD (Voice Activity Detection) 判断"现在有人在说话吗"。它的核心任务是：**在连续的音频流中切出有意义的语音段**。

| 方法 | 原理 | 延迟 |
|------|------|------|
| **能量阈值** | 信号幅度超过阈值 = 在说话 | ~10ms |
| **WebRTC VAD** | 高斯混合模型，6 个频带分析 | ~20ms |
| **Silero VAD** | 轻量神经网络，精确度最高 | ~30ms |

### 工程实践

- **防死等**：设置 0.5s 静音超时，超时自动截断，避免一直等
- **防风扇噪声**：麦克风物理远离风扇，或做频谱减法降噪
- **单行极速刷新**：VAD 检测到语音结束后不等待，立即送入 ASR

---

## 第 2 章：ASR —— 自动语音识别

### 2.1 方案对比

| 方案 | 部署位置 | 延迟 | 精度 | 适用场景 |
|------|---------|------|------|---------|
| **Faster-Whisper** | 本地 Orin NX | 0.5-2s | 高 | 离线/隐私敏感场景 |
| **Whisper (OpenAI API)** | 云端 | 0.3-1s | 极高 | 有网络、多语种 |
| **Gemini Live API** | 云端 | <0.3s | 极高 | 全双工实时对话 |

### 2.2 Ghost Words 幻觉拦截

ASR 在静音段可能输出幻觉文本（"thank you"、"晚安"等）。工程上需要：
- **置信度过滤**：ASR 输出的 confidence < 0.6 直接丢弃
- **语义校验**：识别结果如果是已知的幻觉高频词列表，直接忽略

## 第 3 章：TTS —— 文本转语音

| 方案 | 部署位置 | 延迟 | 音质 |
|------|---------|------|------|
| **VITS** | 本地 NUC（lab 用户） | 0.3-1s | 自然 |
| **Edge-TTS** | 云端 | 0.2-0.5s | 高 |
| **Gemini TTS** | 云端（全双工） | 实时流式 | 极高 |

### 工程实践

- **权限隔离**：TTS 生成的 `.wav` 文件必须用普通用户权限，切忌 `root` 运行（会导致后续文件被锁）
- **声卡独占**：播放前检查 ALSA 设备是否被 PulseAudio 占用，必要时 `pkill pulseaudio`
- **跨机播放**：上位机生成 TTS 文本 → HTTP 发到下位机 → 下位机 `aplay` 播放。注意 `chmod 777` 临时音频文件

---

## 第 4 章：全双工 vs 半双工

| | 半双工 (Push-to-Talk) | 全双工 (Full-Duplex) |
|------|----------------------|---------------------|
| 交互方式 | 按键 → 说话 → 松键 → 等待回复 | 随时说话，随时被打断 |
| 技术方案 | VAD + ASR + LLM + TTS 级联 | WebSocket 长连接 + 流式推理 |
| 代表实现 | 本地 Whisper + Qwen + VITS | Gemini Live API |
| 工程复杂度 | 低 | 高（需处理中断恢复、音频流同步） |

> 全双工的核心挑战：用户说话时机器人也在播放 TTS——需要**回声消除 (AEC)**，否则麦克风会把自己的喇叭声录进去，触发死循环。

---

## 关键词速查

| 术语 | 解释 |
|------|------|
| **VAD** | 语音活动检测，判断是否有人在说话 |
| **ASR** | 自动语音识别，语音→文本 |
| **TTS** | 文本转语音，文本→音频波形 |
| **Ghost Words** | ASR 在静音段的幻觉输出，需置信度过滤 |
| **AEC** | 回声消除，防止喇叭声被麦克风录入 |
| **全双工** | 可随时说话和被打断，无需按键 |
