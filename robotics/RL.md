# 深度强化学习与机器人控制 (Deep Reinforcement Learning in Robotics)

> **架构师引言**：
> 如果说经典动力学（MPC+WBC）是人类工程师一笔一划写出来的“物理法则说明书”，那么强化学习（RL）就是让机器人在无数次摔跤中自己悟出的“肌肉记忆”。在现代足式和人形机器人领域，RL 正在逐渐吞噬原本属于经典控制的底盘领地。

## 一、 核心原理：试错的暴力美学

强化学习的本质不是“教”机器人怎么做，而是给它设定一个**目标（Reward）**，让它在**环境（Environment）**中通过不断试错，自己寻找最优的**策略（Policy）**。

### 1.1 马尔可夫决策过程 (MDP)
一切强化学习都可以抽象为 MDP 框架，这是其底层数学逻辑：
*   **状态 ($S_t$)**：机器人当前“看”到和“感觉”到的世界。例如：关节角度、IMU 加速度、甚至是雷达点云。
*   **动作 ($A_t$)**：机器人决定的输出。例如：目标关节角度、输出到电机的预期扭矩。
*   **奖励 ($R_{t+1}$)**：环境给机器人的反馈。走得稳给 +1，摔倒了给 -100。
*   **策略 ($\pi$)**：大脑（神经网络）。它的工作是看到 $S_t$，输出 $A_t$。

### 1.2 核心思想：延迟满足与价值函数
RL 不仅看眼前的收益，更看重未来的长期收益。算法会计算一个**价值函数 ($Q$值或$V$值)**，用来评估“处于当前状态，并在未来采取最优策略，总共能拿到多少分”。

## 二、 机器人圈的“门派”：主流算法选型

在机器人这类拥有连续物理变量（连续力矩、连续角度）的系统中，传统的离散动作算法（如玩雅达利游戏的 DQN）完全失效。目前统治机器人圈的是 **Actor-Critic (演员-评论家)** 架构。

### 2.1 Actor-Critic 架构直觉
*   **Actor (演员网络)**：负责输出具体的动作（控制电机）。
*   **Critic (评论家网络)**：负责给 Actor 刚才的动作打分（计算 Value）。
像 TD3 这种算法，就是经典的 Actor-Critic 架构。Actor 负责在连续空间中探索路径，而 Critic（在 TD3 中使用两个独立的 Q 网络来缓解高估问题）则负责精准评估这条路径的质量。

### 2.2 主流落地算法对比
| 算法名称 | 核心特点 | 机器人领域的典型应用场景 |
| :--- | :--- | :--- |
| **DDPG** | 深度确定性策略梯度。早期的连续控制王者。 | 相对简单的机械臂单关节控制，目前较少用于复杂全身控制。 |
| **TD3** | DDPG 的进化版，加入双 Q 网络和策略延迟更新。 | 极为稳定的连续控制算法，常用于**移动机器人的路径规划**和避障控制。 |
| **SAC** | 软性 Actor-Critic。引入了“最大化熵”机制，鼓励机器人多去探索未知动作。 | 样本效率极高，适合在**真实机械臂**上直接进行抓取训练。 |
| **PPO** | 近端策略优化。目前**足式/人形机器人领域的绝对统治者**。 | 宇树、ANYmal 等机器狗/人形机器人的步态生成。它限制了每次策略更新的幅度，训练极度稳定。 |

## 三、 在哪“炼丹”：训练环境与仿真平台

深度强化学习极其“贪吃”数据，在物理世界中让一台人形机器人摔几百万次既不现实也破产了。因此，**99% 的 RL 都在仿真器中训练**。

### 3.1 仿真器选型 (Simulators)
*   **Gazebo**：
    *   **定位**：古典且精准。与 ROS 无缝集成。
    *   **特点**：使用 CPU 渲染物理，计算精确但**极度缓慢**。适合验证控制逻辑（如跑通一个 TD3 的路径导航），但不适合需要上亿次迭代的深度强化学习大模型。
*   **MuJoCo**：
    *   **定位**：足式机器人的发源地。
    *   **特点**：使用广义坐标系，动力学求解极其稳定和高效。
*   **Isaac Gym (Nvidia)**：
    *   **定位**：现代 DRL 的“核武器”。
    *   **特点**：利用 GPU **高度并行化**。可以在显存里同时模拟 4096 台机器人一起走路，将原本需要几个月的训练时间压缩到几个小时。

### 3.2 跨越虚实鸿沟 (Sim-to-Real)
在仿真里跑得天下无敌，一上真机当场劈叉，这是 RL 最痛的领悟。为了跨越这个鸿沟，必须使用**域随机化 (Domain Randomization)** 技术：
在 Isaac Gym 中，故意给机器人的质量加上随机误差（比如某条腿突然重了 1kg），故意在关节输出上加入白噪声，故意把地面的摩擦系数调得忽大忽小。只有在这种“地狱随机模式”下存活下来的神经网络，才能抵抗真实世界中电机的齿槽转矩和不平整的水泥地。

## 四、 全链路融合：RL 在系统架构中的生态位

强化学习不是孤立的，它必须和你的上位机感知（DL）和下位机控制（WBC）打交道。目前有两种主流的融合流派：

### 流派一：RL 作为“高级指挥官” (分层架构)
*   **上游**：深度学习 (YOLO/点云) 告诉 RL 前方有个坑。
*   **RL 层**：RL 网络思考后，决定出一个**宏观策略**（比如：跨步距离 0.5 米，期望落足点位置）。
*   **下游**：RL 把这个落足点扔给底层的经典动力学控制器（MPC/WBC）。WBC 负责去计算具体的电机力矩。
*   **评价**：安全可控，出了 Bug 容易排查。大多数商业公司目前处于这个阶段。

### 流派二：RL 吞噬一切 (End-to-End 端到端)
*   **全包裹**：直接把深度相机的深度图（Pixels）和机器人的关节状态喂给一个巨大的 RL 网络。网络的输出直接是几十个电机的目标电流。
*   **评价**：完全抛弃了经典动力学和状态机。上限极高，步态极其自然拟人，但**完全是个黑盒**，调参如同玄学，真机摔了你都不知道是看错了还是算错了。

## 五、 工程实战血泪史：Reward Shaping 的艺术

在 RL 的世界里，**“你怎么定义奖励，它就长成什么样”**。这被称为 Reward Hacking（奖励黑客）。

*   **案例 1**：你让机器人学走路，设定的奖励是“向前移动的距离”。结果机器人发现：向前猛扑重重摔在地上，比小心翼翼迈出半步移动的距离更远。于是它学会了“花式扑街”。
*   **案例 2**：你加入惩罚项“不允许躯干触地”。结果机器人发现只要站在原地不动，就不会触地扣分，也不会扑街。于是它学会了“罚站”。

**工程解法**：优秀的算法工程师本质上是“心理学大师”。一条能让人形机器人走起来的 Reward 函数，通常包含几十项极其微弱的加权（平滑度惩罚、能量消耗惩罚、动作对称性奖励等）。调参的过程，就是与神经网络斗智斗勇的过程。
