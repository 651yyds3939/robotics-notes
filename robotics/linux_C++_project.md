## 项目一：高性能机器人实时数据通信与异步日志中间件

### 背景痛点

在机器人（特别是足式或人形机器人）中，控制循环通常运行在 **1kHz（每秒1000次）** 的极高频率下。如果在控制线程里直接使用 `std::cout` 或者标准的写文件操作，磁盘 I/O 的延迟会导致严重的**内核抖动（Jitter）**，直接导致机器人关节卡顿甚至失控。

### 你要做的“产品”

实现一个专为机器人实时系统设计的、零拷贝的进程间通信（IPC）与异步日志记录系统。

### 核心技术要点（如何体现工业级）

* **环形缓冲区与无锁队列：** 使用现代 C++（C++17/20）和原子操作（`std::atomic`），实现一个单生产者单消费者（SPSC）的**无锁队列**。控制线程只管把 IMU、关节状态数据往队列里“秒扔”，绝不阻塞。
* **异步落盘线程：** 启动一个低优先级的后台线程，专门从队列中读取数据并写入磁盘，支持日志分级（INFO/WARN/ERROR）和自动回滚。
* **共享内存（POSIX Shm）通信：** 实现一个基于共享内存的零拷贝传输模块，让控制进程和视觉/可视化进程之间能以极低延迟共享大块数据（如点云或高频状态量）。
* **运行时零内存分配（Zero Dynamic Allocation）：** 核心循环中严禁出现 `new`、`malloc` 或 `std::vector::push_back`，所有内存必须在系统初始化时一次性分配好（Memory Pool 思想）。

> **面试杀伤力：** “我写过 Linux 串口/网络通信”只能拿 60 分；但如果你说“我针对 1kHz 控制循环实现了一套无锁异步日志系统，将控制线程的抖动控制在 **±10µs** 以内”，这就是 95 分。

## 项目二：轻量级、零动态内存分配的 3D 轨迹规划器（如 S 曲线规划）

### 背景痛点

机器人要伸手抓东西或者抬腿往前走，其关节运动必须平滑。如果速度、加速度或者加加速度（Jerk，跃度）突变，机械结构就会剧烈晃动，甚至烧毁电机。工业界需要极度稳定、边界条件考虑极其完备的轨迹生成算法。

### 你要做的“产品”

用纯 C++ 和 Eigen 矩阵库，写一个独立于任何机器人框架（不依赖 ROS/ROS2 臃肿的外壳）的 **7段式 S 曲线（S-Curve）或时间最优轨迹规划库**。

### 核心技术要点（如何体现工业级）

* **完备的边界条件处理：** 给定起始点、目标点，以及最大速度、最大加速度、最大 Jerk 限制，算法要能自动计算出最优的运动时间剖面。必须完美处理“短距离工况”（即机器人还没加速到最大速度就要开始减速的极端情况）。
* **严格的单元测试：** 利用 **Google Test (GTest)** 编写至少 50+ 个测试用例，覆盖各种正数、负数、零、超出物理限制的边界输入，证明你的代码绝对不会算出 `NaN`（非数）或引发程序崩溃。
* **极致的工程规范：** 整个库采用标准 CMake 构建，对外只暴露极其简洁的 `Init()`、`Evaluate(double time)` 接口，代码通过静态分析工具（如 Clang-Tidy）的严格检查。

> **面试杀伤力：** 这能直接证明你不仅懂机器人运动学/规划算法，更懂得如何把复杂的数学公式落地成高可靠性、高鲁棒性的 C++ 工业级工程代码。

## 项目三：基于行为树（Behavior Tree）的轻量级任务状态调度引擎

### 背景痛点

人形机器人的业务逻辑极其复杂（起立 -> 走路 -> 识别物体 -> 伸手抓取 -> 如果跌倒则触发保护）。如果用传统的 `switch-case` 状态机来写，代码很快就会变成无法维护的“屎山”。目前行业公认的解法是行为树（Behavior Tree）。

### 你要做的“产品”

不依赖开源的大型行为树库，自己纯手工实现一个专为嵌入式/机器人环境优化的**轻量级行为树执行引擎**。

### 核心技术要点（如何体现工业级）

* **多态与面向对象设计：** 设计清晰的 `Node` 基类，派生出 `ControlNode`（Sequence、Selector）和 `ActionNode`、`ConditionNode`。深刻体现你对 C++ 虚函数、多态以及设计模式的理解。
* **异步 Action 支持：** 机器人执行一个“走过去”的动作需要耗时很久，你的引擎必须支持异步的 Action。利用 C++ 的多线程（`std::thread`、`std::future`）或回调机制，实现非阻塞的树结构 Tick 循环。
* **黑板（Blackboard）机制：** 实现一个线程安全的“黑板”数据结构（可以用 `std::unordered_map` 结合互斥锁，或者更高效的方式），用于在不同的行为树节点之间安全地共享机器人状态数据。

> **面试杀伤力：** 绝大多数人只会用现成的行为树插件。如果你能从数据结构、指针管理和线程安全的角度聊清楚一个行为树引擎是如何被调度和 Tick 的，上层业务控制架构师的岗位会非常欢迎你。

## 📂 模块一：指针与内存管理（树的“生命线”）

在 C++ 里，树的节点和节点之间是通过**物理内存地址**连接起来的。这一块通关了，二叉树就成功了一半。

* **1. 指针与引用（Pointer & Reference）：**
* 搞懂 `int* p`（指针）和 `int& r`（引用）的区别。
* 知道什么是 `nullptr`（空指针），以及为什么不能对空指针进行解引用（这是 Linux 报 Segment Fault 的头号杀手）。


* **2. 栈内存 vs 堆内存（Stack vs Heap）：**
* 搞懂为什么普通的变量函数结束就销毁了（栈），而树的节点必须通过动态分配常驻内存（堆）。


* **3. 现代 C++ 智能指针（Smart Pointers）：**
* **核心核心核心：** 彻底抛弃传统的 `new` 和 `delete`。
* 掌握 `std::shared_ptr`（共享智能指针）和 `std::make_shared` 的用法。你后面管理 Kuavo 的行为树节点全靠它。



---

## 📂 模块二：面向对象编程 OOP（树的“骨架”）

行为树的核心思想是“把逻辑抽象成对象”，这一块决定了你代码的架构高级不高级。

* **1. 类与构造函数（Class & Constructor）：**
* 掌握类的封装，理解 `public`（公开）和 `private`（私有）的边界。
* 熟练编写**构造函数初始化列表**（这是现代 C++ 规范，例如：`TreeNode(std::string name) : node_name(name) {}`）。


* **2. 继承（Inheritance）：**
* 搞懂 `class SequenceNode : public TreeNode` 的含义（子类如何继承父类的属性和方法）。


* **3. 虚拟与多态（Virtual & Polymorphism）：**
* **核心核心核心：** 理解虚函数 `virtual` 和重写 `override`。
* 理解为什么可以用一个“父类指针”去指向并运行“子类对象”的方法。这是让总经理（Root）能调度所有基层员工（Action）的底层魔法。



---

## 📂 模块三：C++ 标准模板库 STL（你的“百宝箱”）

写 C++ 不需要手写所有基础数据结构，标准库里有现成的工具可以用。

* **1. 动态数组 `std::vector`：**
* 学会如何声明一个存放指针的数组：`std::vector<std::shared_ptr<TreeNode>>`。
* 掌握最常用的 API：`push_back()`（尾插）、`size()`（大小）、`clear()`（清空）。
* 熟练使用 **基于范围的 for 循环（Range-based for loop）** 来遍历数组。


* **2. 字符串 `std::string`：**
* 用于给你的 Kuavo 节点起名字、打印 debug 日志。



---

## 📂 模块四：现代 C++ 必备小语法（工程里的“润滑剂”）

让你的代码看起来像一个专业系统工程师写出来的，而不是期末大作业。

* **1. 强类型枚举（Scoped Enums）：**
* 掌握 `enum class NodeStatus { SUCCESS, FAILURE, RUNNING };`。
* 用来规范行为树节点的返回状态，避免传统的宏定义冲突。


* **2. 自动类型推导（`auto` 关键字）：**
* 当 C++ 的类名和迭代器长得像老太婆的裹脚布一样时（比如 `std::vector<std::shared_ptr<TreeNode>>::iterator`），一个 `auto` 拯救你的视力。
